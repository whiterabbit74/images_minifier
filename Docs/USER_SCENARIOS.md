# Пользовательские сценарии (обе версии приложения)

Примечание: в коде явно различаются две пользовательские версии/режима.
1) Оконная версия (Dock/полное окно приложения).
2) Менюбар-версия (accessory режим с иконкой в менюбаре).
Ниже все возможные сценарии действий пользователя, которые реализованы в текущем коде.

## Версия 1: Оконная (Dock/полное окно)

### Запуск и базовая навигация
- Запуск приложения и появление основного окна.
- Запуск приложения и восстановление позиции окна (если сохранена).
- Переключение между вкладками `Optimizer`, `Statistics`, `Settings`.
- Переключение темы (Auto -> Dark -> Light -> Auto) кнопкой в тулбаре.
- Открытие настроек через меню приложения (`Settings…`, Cmd+,).
- Открытие информации `About PicsMinifier` через меню приложения.

### Optimizer: загрузка файлов
- Перетаскивание одного файла в окно (drag & drop).
- Перетаскивание нескольких файлов в окно.
- Перетаскивание папки (включая вложенные папки).
- Перетаскивание набора файлов и папок одновременно.
- Перетаскивание неподдерживаемых типов (должны быть проигнорированы, список не заполняется).
- Перетаскивание дубликатов (один и тот же файл в разных путях, ожидание дедупликации).
- Перетаскивание во время активной обработки (должно игнорироваться).
- Перетаскивание когда выбрана вкладка `Statistics` или `Settings` (должен переключиться на `Optimizer`).
- Системное меню `Open Files…` (Cmd+O) и выбор одного файла.
- Системное меню `Open Files…` и выбор множества файлов.
- Системное меню `Open Folder…` и выбор папки.
- Отмена `Open Files…` / `Open Folder…` (без изменений состояния).

### Optimizer: режимы обработки
- `Compress Immediately = true`: сразу старт обработки после дропа/открытия.
- `Compress Immediately = false`: появляется список `Pending`, пользователь вручную нажимает `Start`.
- Нажатие `Start` при наличии `Pending` файлов.
- Нажатие `Start` при отсутствии `Pending` файлов (кнопка не отображается).
- Отображение прогресса (проценты, счетчики, текущий файл).
- Отмена обработки через меню `Cancel` или клавишу `Esc`.
- Завершение обработки с успехом (конфетти, статистика обновляется).
- Завершение обработки с частичными ошибками (статус ошибок, без конфетти).
- Завершение обработки без выигрыша размера (статусы `no-gain` в логике ядра).

### Optimizer: результаты
- Отображение результатов (список файлов, статусы `done/skipped/error/pending/processing`).
- Отображение суммарной панели (число успешных файлов, общий размер сохранения).
- Нажатие `Clear List` и очистка результатов.
- Повторная обработка нового набора после очистки списка.

### Sidebar: пресеты и параметры
- Выбор системных пресетов (`Quality`, `Balanced`, `Saving`).
- Сохранение пользовательского пресета (Save Preset).
- Отмена сохранения пресета (Cancel в алерте).
- Выбор ранее сохраненного пользовательского пресета.
- Смена JPEG качества.
- Смена PNG уровня.
- Смена AVIF качества.
- Смена AVIF усилия.
- Смена WebP качества.
- Смена WebP усилия.
- Переключение GIF режима Lossy/Lossless.
- Включение/выключение дополнительного lossy для GIF.
- Включение/выключение SVG Cleaner.
- Выбор уровня SVG (Safe/Balanced/Max).
- Включение/выключение Resize.
- Ввод собственного размера для Resize.
- Выбор пресета размера (4096/1920/1280/1024/800).
- Выбор режима Resize (Fit/Width/Height).
- Включение/выключение Direct Save (compress immediately).
- Включение/выключение Keep Metadata.
- Включение/выключение Convert to sRGB.
- Выбор Save Mode (Suffix/Separate Folder/Overwrite).
- Предупреждение при выборе `Overwrite`.

### Statistics
- Просмотр статистики текущей сессии.
- Просмотр статистики lifetime.
- Просмотр разбивки по форматам (если данные есть).
- Отображение `No data yet` при отсутствии данных.
- Включение/выключение `Disable Statistics`.

### Settings (простая панель)
- Включение/выключение `Launch at Login`.
- Включение/выключение `Notify on Completion`.
- Включение/выключение `Sound on Completion`.
- Смена темы через сегментированный контрол (Auto/Dark/Light).
- Выбор режима интерфейса (Dock / Menu Bar / Both).
- Смена языка приложения (меню выбора языка).
- Сброс настроек `Reset to Defaults`.

### Системные реакции
- Отправка нотификаций при завершении (если включено).
- Воспроизведение звука при завершении (если включено).
- Dock bounce при завершении.

## Версия 2: Менюбар (accessory режим)

### Базовые сценарии
- Переключение режима интерфейса на `Menu Bar` в настройках.
- Приложение скрывает Dock и остается доступным через менюбар.
- Открытие главного окна через пункт меню `Open App` в менюбаре.
- Открытие настроек через пункт меню `Open Settings…` в менюбаре.
- Завершение приложения через `Quit` в менюбаре.

### Взаимодействие с окном
- Вызов главного окна из менюбара и работа с вкладками и настройками как в версии 1.
- Возврат в менюбар-режим без Dock-иконки после закрытия окна.

### Комбинированный режим
- Выбор `Both` (Dock + Menu Bar) и доступ к менюбару при открытом Dock-режиме.
