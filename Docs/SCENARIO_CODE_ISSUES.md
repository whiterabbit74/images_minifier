# Проблемы, выявленные при анализе сценариев по коду

## 1) Двойная обработка drop-события
- Сценарий: перетаскивание файлов/папок в окно.
- Ожидаемое поведение: один проход обработки.
- Фактический риск по коду: в `ContentView` зарегистрированы два одинаковых `.onDrop`, из-за чего `AppDropDelegate.performDrop` может вызываться дважды.
- Последствия: дублирование списка файлов, двойная обработка, лишние security-scoped ресурсы, неверная статистика.
- Файл: `Sources/PicsMinifierApp/ContentView.swift`.

## 2) Настройки не применяются при открытии файлов через меню
- Сценарии: `Open Files…` и `Open Folder…` через меню приложения (Cmd+O / Cmd+Shift+O).
- Ожидаемое поведение: применяются текущие настройки пользователя (пресеты, параметры, режим сохранения и т.д.).
- Фактический риск по коду: `SessionStore.settingsStore` устанавливается только внутри `AppDropDelegate.performDrop`. Для сценариев выбора файлов через меню `settingsStore` остаётся `nil`, поэтому используется дефолтный `AppSettings`.
- Последствия: компрессия выполняется не с пользовательскими настройками.
- Файлы: `Sources/PicsMinifierApp/ContentView.swift`, `Sources/PicsMinifierApp/SessionStore.swift`.
